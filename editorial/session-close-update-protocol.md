# Session-Close Website Update Protocol

This is a standing Tech-project publication procedure.

Whenever Ashwin asks for Codex commands for a technical website update at the end of a work session, Codex must:

1. Read `editorial/website-update-rules.md`.
2. Read `editorial/checklists/update-review-checklist.md`.
3. Inspect the current website format.
4. Explicitly state before giving the Codex command that the checklist was reviewed.
5. Explicitly summarize what the checklist contains.
6. Incorporate any new publishing preference supplied by Ashwin.
7. Update the canonical policy and checklist when Ashwin says the new preference is permanent or canonical.
8. Then produce the Codex command and task.
9. Treat one session as one local calendar day unless Ashwin explicitly says otherwise.

This procedure requires checking the canonical files. It does not rely on Ashwin remembering to say `Read and follow editorial/website-update-rules.md`.
