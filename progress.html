2026-07-09

WhackSmacker milestone: cleaner content, stricter decks, and Mandarin Chapter 1-5

Today was a big quality pass for WSM. The headline is not just "more content"; it is that the content and package rules are getting much harder to fake. WhackSmacker is still a pre-alpha public development preview, but it is turning into a more usable language-learning package system.

Rules got stricter

The language curriculum rulebook is now the source of truth for future language work. It formalizes pacing for early chapters, requires cumulative vocabulary, and makes strict read-content examples mandatory for normal core review cards. In plain terms: review examples should be real learner-facing sentences from the chapter text, not convenient strings pulled from vocabulary tables, grammar notes, metadata, generated fields, or authoring leftovers.

Review output also got easier to scan because examples now render as clean bullets. Grammar summaries now follow the two-entry shape after each five-chapter block: Grammar -- Easy and Grammar -- Hard.

Korean became the cleanup test case

Korean Chapter 1 was cleaned up as a concrete quality target. The learner-facing title is now Names and First Greetings, authoring clutter was removed, and dialogue names now use target-language forms. That means 마리아, not Maria, while Korean-origin names such as 김민준 stay as full Korean names.

Korean now has Grammar Easy/Hard summaries for Chapters 1-5, 6-10, and 11-15. The terminal UI also got a readability pass: no visible code labels on learner read blocks, pink read/fenced blocks instead of the old very dark block style, display-width table alignment for Hangul/CJK, hidden useless Status clutter, normalized noun notes, a wider scrollable left pane, stable content-pane borders, and clearer menu labels such as Ch N -- title, Han Gul N -- title, and Grammar -- Easy.

Mandarin Chapter 1-5 landed

Chinese Mandarin now has the first five core chapters in both Traditional and Simplified. Traditional was treated as canonical, and Simplified was derived from Traditional so the two tracks stay equivalent except for script.

  • Chapter 1: Greetings and Identity, with 我是N.
  • Chapter 2: Saying My Name, with 我的名字是N.
  • Chapter 3: Asking Names, with 你叫什麼名字？ / 你叫什么名字？.
  • Chapter 4: Introducing This, with 這是N / 这是N.
  • Chapter 5: First Wellbeing Questions, with 你好嗎？ / 你好吗？.

Each chapter has one grammar point, eight read-content lines, eight vocabulary items, and cumulative progression. Traditional includes Traditional characters, Pinyin, and Zhuyin. Simplified includes Simplified characters and Pinyin.

The Mandarin Traditional and Simplified Chapter 1-5 review decks each have 40 entries and 120 cards across English to Chinese, Chinese to English, and Pinyin to Chinese directions. Strict example coverage passed.

Package path kept working

The package feed was rebuilt repeatedly through the package feed repo and checked through ~/bin/wsm-run.sh. Package generation also learned enough about structured Chinese fields to match strict examples against the actual character side instead of the whole pronunciation/meaning block.

Next steps

Next is more day-by-day language content, starting with Japanese Chapter 1-5. Later work should also include a web GUI prototype for display toggles: Traditional versus Simplified Chinese, Zhuyin/Pinyin display, metadata on/off, Korean Hanja on/off, and similar switches. CLI options can follow after the toggle model feels right.
