WackSmacker, Schmeckblacker, GladPecker, or something that actually sells?

WhackSmacker pre-alpha development note

Hello everyone! It's me, your human host. So why the name you ask? Well you know, I just wasn't in the greatest mood the world has ever seen when I picked it, and then just stuck with it because why not. All right now our good friend will take over:

WhackSmacker has reached one of those odd little milestones where it is still absolutely, aggressively pre-alpha, but also starting to feel like a real thing.

The big news is that there is now a Docker image:

sleepiestmario/whacksmacker:latest

That means the app, the package feed, and the first bundled language modules can now travel together in one container. It is not a stable release. It is not something I would tell normal people to trust with their study routine yet. But it is now runnable in a much more predictable way than before, and that matters.

The current recommended Docker run shape is:

docker run --rm -it \
  --user "$(id -u):$(id -g)" \
  -e HOME=/data \
  -v ~/.local/share/whacksmacker:/data \
  sleepiestmario/whacksmacker:latest \
  --data-dir /data \
  --catalogue /feed/catalogue.json

The boring-looking --user and HOME=/data parts are actually important. Without them, Docker can create root-owned progress and data files, which is exactly the sort of tiny irritation that turns into a proper mess later.

The other important change is that the validation VM has been promoted in spirit. It is no longer just “the Docker VM.” It is becoming the WhackSmacker packaging and validation VM.

The current path is:

  1. Docker image first.
  2. Gentoo package validation next.
  3. Arch/AUR after that.
  4. Debian and Ubuntu .deb packages later.
  5. Fedora, openSUSE, and other RPM-family packaging after that.
  6. A generic Linux tarball or AppImage somewhere further down the road.
  7. Maybe Flatpak, Snap, or a desktop wrapper if it still makes sense by then.

That sounds grander than it is. For now it mostly means there is a clean place to build, test, rebuild the feed, rebuild the image, and push it without making the main machine carry all the mess.

The review UI also improved a lot. The review screen now keeps the prompt visible, has a fixed bottom rating bar, and uses the familiar four-answer rhythm:

1 Again
2 Hard
3 Good
4 Easy

Cards now have learner-facing Notes and Example sections too. Internal authoring notes and classification junk should stay hidden, while useful linguistic notes can remain as compact bullets.

The Example section is the next big thing that needs attention. It exists now, and it renders, but the content is not correct enough yet. The goal is not to invent nice-looking example sentences. The goal is to pull one to three literal source sentences from actual read content and attach them to the right vocabulary or review card.

That is tomorrow’s first real task: fix example sentences properly.

The rules for that pass are simple:

  • examples should come from source/read content;
  • examples should be literal lines, not invented replacements;
  • examples should match the card they belong to;
  • examples should render cleanly;
  • conversion-only decks, such as Chinese pronunciation conversion decks, should stay exempt;
  • Japanese remains special until its generated review decks are ready;
  • the renderer should mostly be left alone unless there is a visible output bug.

So, where is WhackSmacker now?

It is not stable.

It is not polished.

It is not ready for a serious public release.

But it now has a package feed, a Docker image, a better review interface, a packaging VM path, and enough structure that the next problems are no longer vague. They are concrete, testable, and annoyingly specific.

That is usually a good sign.
